Understanding Block Elements

Block elements are those that start on a new line and take up the full width available. Here are examples of various block elements:

<div> - Division

The <div> element is a generic container for other elements. It does not inherently represent anything.

<div>This is a division element.</div>

<p> - Paragraph

The <p> element represents a paragraph of text.

<p>This is a paragraph.</p>

<h1> to <h6> - Headings

The <h1> to <h6> elements represent six levels of section headings, <h1> being the highest (or most important) level and <h6> the lowest. 

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>

<ul> - Unordered List

The <ul> element represents an unordered list of items, typically rendered as a bulleted list.

<ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

<ol> - Ordered List

The <ol> element represents an ordered list of items, typically rendered as a numbered list.

<ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

<li> - List Item

The <li> element represents an item in a list. It must be contained in a parent element: an ordered list (<ol>), an unordered list (<ul>), or a menu (<menu>).

<ul>
    <li>Item 1</li>
    <li>Item 2</li>
</ul>

<blockquote> - Block Quotation

The <blockquote> element represents a section that is quoted from another source.

<blockquote>
    This is a block quotation.
</blockquote>

<pre> - Preformatted Text

The <pre> element represents preformatted text, which is typically displayed in a fixed-width font. Whitespace inside this element is displayed as it is, and line breaks are preserved.

<pre>
This is preformatted text.
    It respects whitespace and line breaks.
</pre>

<figure> - Figure

The <figure> element represents self-contained content, like illustrations, diagrams, photos, code listings, etc., that is typically referenced in the main text.

<figure>
    <img src="image.jpg" alt="An example image">
    <figcaption>This is an image caption.</figcaption>
</figure>

<figcaption> - Figure Caption

The <figcaption> element represents a caption or legend for a figure (<figure> element).

<figure>
    <img src="image.jpg" alt="An example image">
    <figcaption>This is an image caption.</figcaption>
</figure>

<hr> - Horizontal Rule

The <hr> element represents a thematic break between paragraph-level elements (e.g., a change of scene in a story, or a shift of topic within a section).

<hr>

<address> - Address

The <address> element represents contact information for its nearest <article> or <body> element ancestor.

<address>
    1234 Street Name<br>
    City, State, ZIP Code
</address>

<article> - Article

The <article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication).

<article>
    <h2>Article Title</h2>
    <p>This is an article.</p>
</article>

<section> - Section

The <section> element represents a standalone section of content which doesn't have a more specific semantic element to represent it.

<section>
    <h2>Section Title</h2>
    <p>This is a section.</p>
</section>

<nav> - Navigation

The <nav> element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

<nav>
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
        <li><a href="#contact">Contact</a></li>
    </ul>
</nav>

<aside> - Aside

The <aside> element represents a portion of a document whose content is only indirectly related to the document's main content.

<aside>
    <p>This is an aside.</p>
</aside>

<header> - Header

The <header> element represents introductory content, typically a group of introductory or navigational aids.

<header>
    <h1>Header Title</h1>
    <p>This is the header section.</p>
</header>

<footer> - Footer

The <footer> element represents a footer for its nearest sectioning content or sectioning root element.

<footer>
    <p>This is the footer section.</p>
</footer>

<main> - Main Content

The <main> element represents the dominant content of the <body> of a document.

<main>
    <h1>Main Content Title</h1>
    <p>This is the main content.</p>
</main>

<details> - Details

The <details> element represents a disclosure widget from which the user can obtain additional information or controls.

<details>
    <summary>More info</summary>
    <p>This is additional information.</p>
</details>

<summary> - Summary

The <summary> element represents a summary, caption, or legend for a <details> element's disclosure box.

<details>
    <summary>More info</summary>
    <p>This is additional information.</p>
</details>
